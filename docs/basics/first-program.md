# 첫 프로그램 만들기

:::{admonition} 이 장에서 배우는 것
:class: note
- 초보자용 가장 안전한 첫 실습 흐름
- Move와 Waypoint를 이용해 간단한 프로그램 만드는 법
- 첫 프로그램에서 반드시 넣어야 할 점 이름과 순서
:::

## 첫 프로그램의 목표

첫 프로그램은 화려할 필요가 없습니다. 목표는 아래 3가지면 충분합니다.

1. 로봇이 **예상 가능한 순서**로 움직인다.
2. 각 점의 의미를 이름으로 설명할 수 있다.
3. 속도를 낮춘 상태에서 공회전 검증이 가능하다.

## 추천 예제: 4점 이동 프로그램

처음에는 아래 구조가 가장 좋습니다.

```text
home_safe
approach_work
work
retreat
```

- `home_safe`: 작업 시작 전/후 대기 위치
- `approach_work`: 작업점으로 들어가기 전 안전 접근점
- `work`: 실제 작업점
- `retreat`: 작업 후 빠져나오는 점

이 구조를 먼저 익히면 나중에 pick/place, 삽입, 검사 공정으로 확장하기 쉽습니다.

## 실제 진행 순서

### 1. 작업 공간 정리

- 주변에 사람이 없는지 확인
- 공구와 작업대 사이 거리를 확인
- 첫 실행은 반드시 저속으로 진행

### 2. 로봇 초기화

- 전원 상태 확인
- 로봇 arm 초기화
- 필요 시 브레이크 해제
- Move 화면에서 현재 자세 확인

### 3. 새 프로그램 만들기

- 새 Program 생성
- 필요하면 새 Installation 저장
- 파일명에 날짜/버전/설비명을 넣기

예: `cellA_first_program_v01`

### 4. Move 노드 추가

가장 먼저 Move 노드를 하나 추가합니다. 이때 첫 Waypoint가 함께 만들어집니다.

### 5. 첫 점 `home_safe` teaching

- 다른 장비와 충분히 떨어진 위치
- 시야가 좋고 복귀하기 쉬운 위치
- 관절 자세가 무리 없는 위치

### 6. 두 번째 점 `approach_work` teaching

- 작업점 바로 위 또는 앞
- 충돌 없이 접근 가능한 위치
- 실제 작업 자세로 들어가기 전 중간점

### 7. 세 번째 점 `work` teaching

- 실제 작업 대상 위치
- 툴 방향이 정확해야 함
- TCP와 Feature가 올바른지 확인 필요

### 8. 네 번째 점 `retreat` teaching

- 작업 후 바로 빠져나오는 점
- 보통 `approach_work`와 유사하거나 동일한 경우도 많음

## 첫 프로그램에서 MoveJ와 MoveL을 어떻게 섞을까

입문자는 아래처럼 시작하는 것을 권장합니다.

- `home_safe -> approach_work`: **MoveJ**
- `approach_work -> work`: **MoveL**
- `work -> retreat`: **MoveL**
- `retreat -> home_safe`: **MoveJ**

이 조합은 "멀리서는 관절 이동으로 빠르게 자세를 만들고, 작업 근처에서는 선형 이동으로 제어한다"는 감각을 익히기에 좋습니다.

## 점 이름은 꼭 의미 있게

좋지 않은 이름:

- `point1`
- `aaa`
- `test2`

좋은 이름:

- `home_safe`
- `pick_approach`
- `pick`
- `place_approach`
- `place`

:::{tip}
문서와 프로그램 이름을 맞추면 교육이 쉬워집니다. 예를 들어 매뉴얼 스크린샷에도 `home_safe`라는 이름이 보이게 만들어 두세요.
:::

## 첫 실행 절차

1. 속도 슬라이더를 충분히 낮춘다.
2. 공구가 작업물에 닿지 않는 높이에서 먼저 공회전한다.
3. 각 점에서 정지해 자세가 맞는지 본다.
4. 이상 없으면 실제 높이로 조금씩 내려간다.

## 자주 하는 실수

### `work` 점부터 직접 잡음
접근점 없이 바로 작업점으로 들어가면 충돌 위험과 teaching 난이도가 같이 올라갑니다.

### 점 이름을 나중에 바꾸려고 미룸
나중에는 점 수가 늘어나서 더 귀찮아집니다. 처음부터 의미 있게 짓는 편이 훨씬 좋습니다.

### TCP 확인 없이 MoveL 사용
MoveL이 기대와 다르게 보이면 먼저 TCP를 의심해야 합니다.

### speed만 낮추고 자세 검증은 안 함
느리게 움직여도 방향이 틀리면 결국 문제입니다. 자세와 기준 좌표를 함께 봐야 합니다.

## 실습 과제

- 같은 프로그램을 `pick -> place` 구조로 바꿔 보기
- `retreat`를 삭제했을 때 왜 불안해지는지 비교해 보기
- `home_safe`를 두 개 만들어 `home_left`, `home_right`로 나눠 보기
