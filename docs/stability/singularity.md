# 특이점

:::{admonition} 이 장에서 배우는 것
:class: note
- 특이점이 무엇인지
- UR에서 자주 만나는 세 가지 대표 특이점
- 프로그램과 설비 배치로 어떻게 회피할지
:::

## 특이점을 초보자용으로 설명하면

특이점은 로봇이 **그 자세 근처에서 자연스럽게 움직이기 어려워지는 위치/자세**입니다. 이 구간에 가까워지면 로봇이 멈추거나, 갑자기 빨라지거나, 자세가 이상하게 바뀌는 느낌을 줄 수 있습니다.

즉, "목표점은 맞는데 가는 길이 불안한 상태" 라고 이해하면 쉽습니다.

:::{warning}
특이점 근처에서는 예기치 않은 빠른 움직임이나 정지가 발생할 수 있으므로, 실제 장비 검증은 항상 저속과 충분한 이격거리에서 진행하세요.
:::

## UR에서 먼저 알아둘 세 가지

### 1. Outer workspace limit
로봇이 너무 멀리 뻗으려 하거나 최대 작업영역 바깥쪽에 가까워질 때 생기는 문제입니다.

### 2. Inner workspace limit
로봇 베이스 바로 위/아래 중심 원통 부근에서 생기는 문제입니다. 작업 대상이 베이스 축에 너무 가깝게 놓이면 자주 체감됩니다.

### 3. Wrist alignment
손목 축 정렬이 나빠지는 자세에서 생기는 문제입니다. 특히 공구 방향을 억지로 맞추려 할 때 자주 나타납니다.

## 초보자가 이해하기 쉬운 증상

- 같은 점인데 어떤 방향으로 접근하면 잘 되고, 어떤 방향에서는 안 됨
- MoveL로 가다가 속도가 이상하게 느껴짐
- 손목이 불필요하게 많이 회전함
- 베이스 바로 위 작업에서 자세가 자꾸 꼬임

## 각각 어떻게 피할까

### Outer workspace limit 회피

- 작업대를 로봇에 조금 더 가깝게 배치
- 끝점에 여유를 두고 접근점 생성
- 최대 reach 근처에서 작업하지 않도록 배치 변경

### Inner workspace limit 회피

- 베이스 중심 원통을 피하도록 작업물 위치 이동
- 작업면을 옆으로 약간 빼기
- 상하 방향으로만 드나드는 구조를 피하고 접근 각도 추가

### Wrist alignment 회피

- 공구 방향을 약간 오프셋
- 손목 정렬이 심해지지 않도록 중간 자세 변경
- 먼 이동은 MoveJ, 작업 근처만 MoveL로 분리

## 프로그램 차원에서의 회피 전략

### 전략 1. 접근점을 늘린다
특이점은 보통 "작업점 자체"보다 "그 점으로 가는 경로"에서 문제를 일으킵니다. 중간점을 추가하면 자세를 풀 수 있습니다.

### 전략 2. 큰 자세 변화는 MoveJ로 뺀다
멀리서 자세를 바꿔 놓고, 작업 근처에서만 MoveL을 쓰면 훨씬 안정적일 수 있습니다.

### 전략 3. 작업물을 기준으로 Feature를 다시 생각한다
테이블이나 지그를 살짝 회전/이동하는 것만으로도 특이점이 줄어드는 경우가 많습니다.

## 설비 차원에서의 회피 전략

- 로봇 설치 높이 변경
- 작업대 위치 이동
- 로봇 방향 변경
- 공구 길이/각도 조정
- 작업 대상 방향 재설계

초보자는 종종 프로그램만 고치려 하지만, 실제로는 **설비 배치가 더 큰 원인**일 때가 많습니다.

## 실습: 특이점 감각 익히기

### 실습 A. 베이스 중심 근처 점 비교

- 베이스에서 가까운 점과 옆으로 100 mm 이동한 점을 비교
- 어떤 방향이 더 자세가 자연스러운지 기록

### 실습 B. 접근 방향 바꾸기

- 같은 작업점에 대해 접근점 방향을 2~3개 바꿔 보기
- 손목 회전이 얼마나 달라지는지 확인

### 실습 C. MoveJ / MoveL 역할 분리

- 큰 이동은 MoveJ
- 작업 근처만 MoveL

이 구조가 특이점 회피에 도움이 되는지 비교

## 자주 하는 실수

### 작업점을 절대 못 바꾸는 값으로 생각함
실제로는 접근 방향, 자세, 작업대 위치를 바꾸면 해결되는 경우가 많습니다.

### MoveL만 고집함
공정상 정말 필요한 구간만 MoveL로 남기는 편이 안정적일 수 있습니다.

### 손목 회전을 디버깅하지 않음
실제 문제는 목적지가 아니라 손목 정렬일 수 있습니다.
