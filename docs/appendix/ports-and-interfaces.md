# 포트와 인터페이스 요약

이 페이지는 통신을 시작할 때 빠르게 참고하는 **포트/용도 치트시트**입니다. 실제 적용 전에는 반드시 사용 중인 소프트웨어 버전의 최신 공식 문서를 함께 확인하세요.

## 대표 인터페이스

| 인터페이스 | 대표 포트 | 용도 | 초보자 메모 |
| --- | --- | --- | --- |
| Dashboard Server | 29999 | 프로그램 로드/재생/정지, 상태 확인 | 먼저 연습하기 가장 쉬움 |
| Primary | 30001 | 상태 + 메시지, URScript 명령 | 동적 제어/상태 확인용 |
| Secondary | 30002 | 상태 + URScript 명령(간소) | Primary보다 단순하게 접근 |
| Realtime | 30003 | 레거시 실시간 계열 | 신규 설계는 RTDE 우선 검토 |
| RTDE | 30004 | 구조화된 주기적 데이터 교환 | 상태 모니터링에 적합 |
| Primary read-only | 30011 | 읽기 전용 상태 데이터 | 읽기 전용이 필요할 때 |
| Secondary read-only | 30012 | 읽기 전용 상태 데이터 | 단순 모니터링 |
| Realtime read-only | 30013 | 읽기 전용 실시간 계열 | 레거시/특수 목적 |
| Interpreter | 30020 | 인터프리터 계열 | 고급 원격 제어용, 초보자는 후순위 |

## 초보자용 선택 규칙

- 프로그램 실행/정지부터 해보고 싶다 -> **Dashboard**
- 로봇과 외부 서버가 문자열/숫자 주고받게 하고 싶다 -> **Socket**
- 상태를 주기적으로 안정적으로 읽고 싶다 -> **RTDE**
- URScript를 외부에서 보내고 싶다 -> **Primary / Secondary 계열**

## 문서에 꼭 같이 적을 것

포트 번호만 적으면 나중에 해석이 어려워집니다. 아래 항목을 함께 적으세요.

| 항목 | 예시 |
| --- | --- |
| 로봇 IP | 192.168.0.10 |
| 상위 PC IP | 192.168.0.20 |
| 포트 | 29999 |
| 방향 | PC -> Robot |
| 프로토콜 | TCP |
| 데이터 | play / stop / pose / 상태값 |
| 단위 | m, rad, bool |
| 대상 버전 | PolyScope 5 / PolyScope X |

## 주의 사항

### Dashboard와 RTDE는 용도가 다르다
Dashboard는 간단한 제어 명령, RTDE는 구조화된 주기 데이터 교환 쪽이 더 핵심입니다.

### Realtime 인터페이스는 레거시 성격을 먼저 의심하자
새 프로젝트에서는 RTDE부터 검토하는 편이 일반적으로 더 이해하기 쉽습니다.

### PolyScope X는 별도 확인 필요
Dashboard 개념 대신 Robot API 등 버전 차이가 있을 수 있으므로, 내부 문서에 대상 버전을 명확히 적어 두세요.

## 우리 팀용으로 확장하면 좋은 표

| 시스템 | 인터페이스 | 포트 | 비고 |
| --- | --- | --- | --- |
| MES PC | Dashboard | 29999 | 시작/정지용 |
| Vision PC | Socket | 5000 | 좌표 송신 |

:::
