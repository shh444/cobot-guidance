## TCP 란 무엇인가?

TCP(Tool Center Point)는 **로봇 플랜지에 장착된 엔드이펙터의 작업 기준점**입니다.

즉, 로봇 팔 끝 금속 플랜지가 중요한 것이 아니라, 실제 작업에서는 TCP가 중요합니다.

TCP를 조금 더 정확히 말하면, TCP는 **툴 좌표계 기준 X/Y/Z 거리(오프셋)**로 정의되는 점입니다.
보통 UR에서는 TCP를 `p[x, y, z, rx, ry, rz]` 형태로 다루며,
`x, y, z`는 위치 오프셋(거리), `rx, ry, rz`는 공구 자세(방향)를 의미합니다.

```{figure} ../_static/tcp-definition-illustration.png
:alt: TCP의 위치(X,Y,Z)와 자세(RX,RY,RZ) 값을 설명하는 개념도
:width: 68%

TCP는 위치(X,Y,Z)와 자세(RX,RY,RZ)로 정의되며, 실제 공구 끝 작업점을 기준으로 설정합니다.
```

예를 들면:

- 그리퍼 팁의 중간
- 흡착 패드 중심
- 스크류 드라이버 비트 끝
- 디스펜서 노즐 끝

```{figure} ../_static/tcp-concept-diagram.svg
:alt: TCP 개념도
:width: 100%

TCP는 로봇 플랜지가 아니라 실제 공구 끝 작업점을 기준으로 설정합니다.
```

## 왜 TCP가 중요한가

TCP는 단순한 설정값이 아니라, 로봇의 위치/자세 계산과 안전 판단의 기준점입니다.

1. 웨이포인트 저장 기준은 TCP입니다.  
   로봇이 티칭으로 저장하는 웨이포인트는 플랜지 기준이 아니라 TCP 기준의 pose입니다.  
   따라서 TCP가 달라지면 같은 웨이포인트라도 실제 경로와 자세가 달라집니다.

2. 자세 변경(RX/RY/RZ)은 TCP 기준으로 적용됩니다.  
   예를 들어 `RX +30`을 주면 TCP를 기준으로 공구 자세가 회전합니다.  
   TCP가 정확해야 의도한 접근 각도와 방향을 재현할 수 있습니다.

3. 접촉 공정 계산의 기준점도 TCP인 경우가 많습니다.  
   용접 터치센싱, 샌딩 각도 보정, 삽입/스크류 공정의 접촉 위치 계산은 TCP를 기준으로 수행되는 경우가 많습니다.

4. 안전 계산에서도 TCP가 핵심 기준점입니다.  
   예를 들어 스크류 드라이빙에서는 비트 끝이 가장 위험한 접촉 지점이므로, 해당 지점을 TCP로 정의해야 실제 위험을 반영해 안전 조건을 설정할 수 있습니다.

즉 TCP는 안전과 어플리케이션 프로그래밍 양쪽에서 모두 중요하게 사용됩니다.

## Tool Flange와 TCP의 차이

| 항목 | 의미 |
| --- | --- |
| Tool Flange | 로봇암 끝의 회색 금속 플랜지 끝단부 |

## TCP teaching 실무 팁

### 서로 다른 자세에서 측정하기
한 방향에서만 비슷하게 잡으면 계산 품질이 떨어집니다. 서로 충분히 다른 각도에서 접근하는 것이 좋습니다.

### 세 점보다 네 번째 검증점을 추가하기
계산은 최소 점 수로 가능하더라도, 검증용 한 점을 더 쓰면 오류를 빨리 잡을 수 있습니다.

### 툴 교체 후에는 이름만 복사하지 않기
툴 형상이 바뀌면 기존 TCP 이름을 그대로 재사용하지 않는 편이 좋습니다.

:::{tip}
사내 문서에는 TCP 표를 따로 두세요. `TCP 이름 / 용도 / 기준점 사진 / 최근 검증일 / 관련 프로그램` 형태가 좋습니다.
:::

## TCP 정확도 검증: 회전 중심 확인 (GIF)

TCP는 자세각(RX/RY/RZ) 계산의 기준점이므로, TCP 값이 정확하면 공구는 **TCP 점 기준**으로 회전합니다.  
반대로 TCP 값이 잘못되면 공구가 TCP가 아닌 다른 점을 중심으로 회전하며, 팁이 원호처럼 크게 흔들립니다.

```{figure} ../_static/tcp-validation-rotation-center.gif
:alt: 잘못된 TCP 값일 때 TCP 점이 아닌 다른 점 기준으로 회전하는 검증 예시
:width: 100%

회전 중심이 TCP에서 벗어나면 TCP 입력값이 실제 공구 기준과 어긋났을 가능성이 큽니다.
```

### 어떻게 확인할까

1. 안전 높이에서 로봇을 정지하고 TCP 표시를 켭니다.
2. X/Y/Z 위치는 유지한 채 RX/RY/RZ만 소각도(예: ±10~20도)로 변경합니다.
3. 공구 팁(내가 정의한 TCP 점)의 움직임을 관찰합니다.

- 정상: TCP 점이 기준점 근처에서 유지되고, 공구 자세만 바뀌는 느낌으로 회전
- 비정상: TCP 점이 원을 그리며 크게 이동하거나, 다른 점을 중심으로 도는 움직임

이 검증으로 TCP 입력이 맞는지 빠르게 판단할 수 있습니다. 비정상이면 TCP를 다시 teaching한 뒤 같은 검증을 반복하세요.

## 여러 TCP를 쓸 때의 기본 전략

툴 하나에 실제 작업점이 여러 개일 수도 있습니다. 예를 들면 듀얼 그리퍼나 길이가 다른 팁입니다.

이때 초보자에게는 다음 전략을 권장합니다.

- TCP를 목적별로 명확히 나눈다
- 프로그램 상단에 현재 어떤 TCP를 쓰는지 주석/설명 추가
- 먼저 하나의 TCP로 안정화한 뒤 여러 TCP로 확장

## 자주 하는 실수

### TCP 위치는 맞는데 방향을 신경 안 씀
삽입, 스크루, 비전 정렬 작업은 방향이 특히 중요합니다.

### TCP를 잡고도 Feature는 기본값 그대로 둠
TCP와 Feature는 같이 생각해야 합니다. 공구 기준과 작업물 기준이 둘 다 맞아야 합니다.

### 툴 교체 후 Payload만 바꾸고 TCP는 안 바꿈
질량과 기준점은 다른 문제입니다. 둘 다 확인해야 합니다.
