# 좌표계와 Feature

:::{admonition} 이 장에서 배우는 것
:class: note
- Base / Tool / Feature를 어떻게 구분할지
- Point / Line / Plane Feature를 언제 쓰면 좋은지
- 작업물을 기준으로 프로그램을 짜는 사고방식
:::

## 왜 좌표계가 필요한가

로봇은 항상 어떤 기준을 바탕으로 위치를 해석합니다. 초보자는 보통 "X, Y, Z는 하나뿐"이라고 생각하기 쉬운데, 실제로는 **어떤 좌표계를 선택했느냐**가 매우 중요합니다.

## 가장 먼저 이해할 3가지 좌표

### Base
로봇 베이스를 기준으로 한 좌표입니다. 가장 기본입니다.

### Tool
현재 활성 TCP를 기준으로 한 좌표입니다. 공구가 보는 방향 기준으로 이동할 때 유용합니다.

### Feature
작업물, 테이블, 지그, 컨베이어 같은 **외부 대상**을 기준으로 한 좌표입니다.

## Feature를 초보자용으로 설명하면

Feature는 "작업물 쪽 기준 좌표" 입니다.

예를 들면:

- 테이블 모서리
- 지그 기준면
- 컨베이어 진행 방향
- 비전 좌표 원점

이 개념이 중요한 이유는, 작업물이 약간 이동해도 Feature만 조정하면 프로그램 전체를 다시 teaching하지 않아도 될 수 있기 때문입니다.

## Point / Line / Plane Feature

| 종류 | 의미 | 추천 상황 |
| --- | --- | --- |
| Point | 한 점 기반 | 단순 기준점, 홈 기준, 간단한 참조 |
| Line | 두 점 기반 방향 포함 | 컨베이어 방향, 긴 기준선 |
| Plane | 평면 기반 좌표계 | 테이블, 지그, 비전 작업면 |

실무에서는 보통 **Plane Feature** 가 가장 활용도가 높습니다. 작업면과 방향을 함께 설명하기 좋기 때문입니다.

## 초보자용 직관

- TCP는 "공구 끝이 어디인가"
- Feature는 "작업물이 어디에 놓여 있나"

라고 이해하면 좋습니다.

## Plane Feature를 추천하는 이유

Plane Feature는 점 하나보다 **방향 정보를 더 안정적으로 표현**할 수 있습니다. 테이블 작업, 트레이 작업, 검사 작업처럼 면이 있는 공정에서는 특히 유용합니다.

## 실습: 테이블 기준 Plane Feature 만들기

### 목표
작업대가 조금 이동해도 Feature만 수정해서 프로그램을 맞출 수 있는 구조 만들기

### 권장 절차

1. 테이블 모서리 또는 기준 핀을 정한다.
2. 원점으로 쓸 점을 먼저 정한다.
3. X 방향을 정의할 점을 정한다.
4. Y 방향을 정의할 점을 정한다.
5. 만든 Plane Feature 이름을 `table_plane`처럼 의미 있게 저장한다.
6. 작업 Move 노드의 기준 Feature를 `table_plane`으로 지정한다.

## Feature를 쓰면 좋은 대표 상황

### 같은 설비를 여러 셀에 복제할 때
셀마다 작업대 위치가 조금 달라도 Feature로 보정하기 쉬움

### 작업물이 약간 이동할 수 있을 때
프로그램 전체 재teaching보다 Feature 수정이 효율적

### 비전 시스템 좌표를 연결할 때
비전이 주는 기준 좌표를 작업물 frame과 연결하기 좋음

## Move 화면과 Feature의 관계

Move 화면에서 현재 선택한 기준 좌표가 무엇인지 항상 확인하세요.

같은 X 방향 이동이라도:

- Base 기준 X
- Tool 기준 X
- Feature 기준 X

는 전혀 다른 움직임처럼 보일 수 있습니다.

:::{important}
초보자가 "분명 오른쪽으로 조그했는데 왜 이상한 데로 가지?"라고 느낄 때, 대부분은 기준 좌표가 예상과 다릅니다.
:::

## 자주 하는 실수

### 작업물 기준 작업인데 Base로만 teaching함
처음에는 되지만, 작업대 위치가 바뀌면 다시 전부 잡아야 합니다.

### Plane은 만들었지만 Move 노드가 여전히 Base 기준임
Feature를 만들어 놓고 실제 프로그램에서 참조하지 않으면 효과가 없습니다.

### Feature 이름이 불명확함
`plane1`, `plane2`보다 `table_plane`, `trayA_plane`이 훨씬 좋습니다.

## 초보자용 베스트 프랙티스

- 작업물에 기준면이 있다면 Base보다 Feature를 먼저 고려
- 점 하나로 될 것 같아도 가능하면 Plane/Line을 검토
- Feature 이름에 대상 물체명을 넣기
- Program과 문서에 "이 Move는 어떤 Feature 기준인가"를 표시

## 체크리스트

- [ ] Base / Tool / Feature 차이를 설명할 수 있다
- [ ] Plane Feature가 왜 유용한지 이해했다
- [ ] Move 화면에서 현재 기준 좌표를 확인할 수 있다
- [ ] 작업물 중심 사고로 프로그램을 짤 수 있다
