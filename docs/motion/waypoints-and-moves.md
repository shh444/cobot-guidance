# 웨이포인트와 이동 명령어

:::{admonition} 이 장에서 배우는 것
:class: note
- Waypoint가 Move와 어떤 관계인지
- Fixed / Relative / Variable Waypoint 차이
- MoveJ / MoveL / MoveP를 언제 쓰면 좋은지
:::

## 로봇이 이동하는 방식

UR 프로그램에서 로봇을 움직이는 가장 기본은 **Move + Waypoint** 입니다.

쉽게 말하면:

- **Move** 는 "어떤 방식으로 움직일지"
- **Waypoint** 는 "어디로 갈지"

를 의미합니다.

Waypoint의 경우 1) 로봇의 각 관절들[J0-J5], 2) TCP의 위치p[x,y,z,rx,ry,rz]로 기억이 되며 또한 **경로(trajectory)** 도 중요합니다.  
같은 시작점과 도착점이라도 Move 종류에 따라 중간 경로(trajectory)는 크게 달라집니다.

예를 들어 같은 목적지라도:

- MoveJ면 관절 자세 중심으로 이동하고
- MoveL이면 TCP가 선형(직선)으로 이동하며
- MoveP면 공정용으로 부드럽고 연속적인 경로를 만들 수 있습니다.

## Waypoint의 3가지 기본 타입

### 1. Fixed Waypoint

가장 먼저 배우는 기본 방식입니다. 조그/프리드라이브로 로봇을 원하는 위치로 가져간 뒤 그 자세를 저장합니다.
즉 내가 저장한 위치 이며 가장 기본 값 입니다.

**추천 상황**

- 홈 위치
- 접근 위치
- 지그 고정 위치
- 교육용 첫 프로그램

### 2. Relative Waypoint

직전 위치를 기준으로 얼마나 더 이동할지를 나타냅니다. 절대 위치 teaching보다 "이전 점에서 얼마만큼 이동"이라는 사고가 필요합니다.

**추천 상황**

- 같은 패턴을 일정 간격으로 반복할 때

### 3. Variable Waypoint

목표 위치가 변수로 결정되는 방식입니다. 비전 보정, 외부 데이터 수신, 사용자 입력에 따라 목표가 달라질 때 쓰기 좋습니다.

보통 pose 변수 형태로 사용합니다.

```text
calculated_pose = p[0.400, -0.100, 0.250, 3.1416, 0.0, 0.0]
```

:::{important}
Variable Waypoint는 매우 유연하지만, 기준 Feature와 단위를 잘못 이해하면 오히려 실수가 커집니다. 처음에는 고정점 teaching을 충분히 익힌 뒤 쓰는 것이 좋습니다.
:::

## MoveJ 명령어 이해하기

MoveJ는 현재 관절 각도에서 목표 관절 각도로 한 번에 보간해 이동합니다.  
그래서 **TCP의 중간 경로(trajectory)는 직선이 아닐 수 있고**, 경우에 따라 크게 휘는 경로가 나올 수 있습니다.

이 특성 때문에 MoveJ는 작업점에서 충분히 떨어진 구간을 빠르게 이동할 때 특히 유리합니다.

보통 다음처럼 시작하면 좋습니다.

- **직선 이동이 필요 없는 경우**
- 홈 위치로 이동
- Inner/Wrist 특이점을 회피하고 싶은 경우
- 자세 크게 변경

## MoveL 명령어 이해하기

MoveL은 **경로가 직선으로 움직이도록** 계산됩니다. 

MoveL이 유용한 경우:

- 구멍 삽입
- 작업물 표면을 따라 이동
- 박스를 내려놓음

## MoveP 명령어 이해하기

MoveP는 MoveL과 같은 직선 이동 이지만 모든 웨이포인트에 블랜드가 포함되어 해당 지점에 도착하지 않고 우회해서 움직입니다.
즉 모든 구간에서 동일한 속도를 유지합니다.

- 공정 속도를 일정하게 유지하고 싶을 때
- 점과 점 사이를 매끄럽게 이어야 할 때
- 디스펜싱/도포처럼 "끊김 없는 경로"가 중요할 때

:::{important}
MoveP는 항상 속도가 일정합니다. 다만 예를 들어 로봇이 위로 올라갔다가 아래로 내려가는 모션을 반복한다면 기구적으로 관절이 한 방향으로 돌다가 다시 반대 방향으로 움직여야 합니다. 그렇게 되면 물리적인 이유로 속도를 일정하게 움직일 수 없으며 제어가 정상적으로 되지 않아 로봇의 모션이 이상해질 수 있습니다. 즉 MoveP는 경로에 주의를 하여 사용해야 합니다.
:::
